🥔 避雷观察薯 
一句话定义：基于小红书原生避雷场景的风险分级决策工具，将“情绪化吐槽”转化为“结构化决策坐标”。

🎯 项目核心价值 (Product Value)
在小红书“万物皆可避雷”的生态下，用户面临决策瘫痪。本工具旨在解决：

痛点：用户搜索“XX 避雷”时已有明确意向，但被过载且无差别的负面信息阻断决策。

方案：通过 AI 语义降噪，将避雷帖分为 【痒点、中雷、大雷、致命雷】 四级。

目标：帮用户看清雷点是否可以接受，帮达人产出真诚内容，帮品牌“听劝”改进。

🛠️ 技术实现 (Tech Stack)
核心引擎: Perplexity AI (Sonar-Reasoning) - 负责联网搜索与非结构化语义抓取。

前端交互: Streamlit - 快速部署响应式 Web 应用。

数据可视化: Plotly - 构建交互式五维避雷雷达图。

逻辑模型: 独创“痛痒分级判定模型”，通过 Prompt Engineering 实现从事实提取到风险影响评估的转化。

🧠 产品深度思考 (Insights)
1. 为什么必须用 AI？
语义降噪：避雷帖中存在大量情绪化噪音，AI 能从“退钱！”、“避雷！”等词汇中提取真实的业务指标（如：售后流程冗长、特定场景适配问题）。

逻辑对齐：将不同用户的个性化表达，强制映射到【品质、性价比、售后、营销、耐用度】五个标准维度。

2. 动手后的新启发
从“避雷”到“匹配”：在开发过程中发现，避雷帖其实是最好的“预期管理”工具。产品不应只是劝退，而是告诉用户：“这个产品的缺点，你这种性格/预算的人能不能忍？”

商业闭环：品牌看板模块展示了将“负评”转化为“改进建议”的可能性，它是小红书“听劝文化”的数字化落地。

🚀 关键设计决策
角色化视图：同一套数据，为消费者提供决策、为达人提供选题视角、为品牌提供整改看板。

高频避雷信号：增加标签云模块，确保用户能瞬间捕捉颗粒度极细的“真实证据”。
